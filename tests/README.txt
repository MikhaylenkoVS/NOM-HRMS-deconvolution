========================================================================
tests/ — автоматические тесты проекта NOM-HRMS-deconvolution
========================================================================

Назначение
----------
Каталог содержит набор автоматических тестов (pytest), проверяющих
корректность и надёжность конвейера деконволюции масс-спектров ПОВ.
Тесты гарантируют, что изменения кода не нарушают химическую и
алгоритмическую логику метода (назначение формул, поиск изотопных серий,
структурная валидность результатов).

Состав
------
  unit/         Модульные тесты отдельных функций ядра:
                  test_denoise.py                — очистка спектра от шума;
                  test_assign_formulas.py        — назначение брутто-формул;
                  test_find_series.py            — поиск серий (–COOH/–OH);
                  test_chemical_validity.py      — химическая корректность;
                  test_structural_validity.py    — валидность структур;
                  test_annotations_consistency.py— согласованность разметки.
  integration/  Интеграционные тесты полного конвейера:
                  test_pipeline_integration.py   — сквозной прогон метода;
                  test_app_smoke.py              — «smoke»-прогон приложения
                                                   по всем тестовым наборам.

Запуск
------
Из корня репозитория:

    pytest

Конфигурация pytest задана в файле pytest.ini (pythonpath = ., testpaths =
tests). Тесты используют синтетические наборы данных из data/test_sets/.

Связь с общим конвейером
------------------------
Тесты — контур контроля качества метода. Модульные тесты покрывают
отдельные шаги обработки спектров (src/core), а интеграционные проверяют
их совместную работу и воспроизводимость результатов на эталонных
синтетических данных.
