# tests/ — автоматические тесты проекта NOM-HRMS-FGA

## Назначение

Автоматические тесты (pytest, 105 шт.) проверяют корректность конвейера
деконволюции масс-спектров ПОВ. Тесты гарантируют, что изменения кода не
нарушают химическую и алгоритмическую логику метода.

## Состав

### unit/ — модульные тесты

| Файл | Проверка |
|------|----------|
| `test_core_utils.py` | _ppm_error, parse_formula, нормализация brutto, exact_mass (66 тестов) |
| `test_van_krevelen.py` | Диаграмма Ван-Кревелена, классификация NOM-областей (17 тестов) |
| `test_find_series.py` | Поиск гомологических серий, max_consecutive_misses (6 тестов) |
| `test_structural_validity.py` | Структура CSV-файлов (4 теста) |
| `test_app_fallback.py` | Консистентность сигнатур embed_figure (3 теста) |
| `test_nom_prioritize.py` | NOM-приоритизация формул (2 теста) |
| `test_denoise.py` | Шумоподавление: recall сигналов и шума |
| `test_assign_formulas.py` | Назначение брутто-формул |
| `test_chemical_validity.py` | Химическая корректность данных |
| `test_annotations_consistency.py` | Согласованность аннотаций |

### integration/ — интеграционные тесты

| Файл | Проверка |
|------|----------|
| `test_pipeline_integration.py` | Сквозной прогон: denoise → assign → series |
| `test_app_smoke.py` | Smoke-прогон приложения по всем тестовым наборам |

## Запуск

Из корня репозитория:

```bash
pytest tests/ -q            # все тесты (~2 мин)
pytest tests/ -q -m unit    # только unit-тесты
pytest tests/ -q -m smoke   # только быстрые smoke-тесты
```

Конфигурация pytest задана в файле `pytest.ini`. Тесты используют синтетические
наборы данных из `data/test_sets/`.

## Связь с общим конвейером

Тесты — контур контроля качества метода. Модульные тесты покрывают отдельные шаги
обработки спектров (`src/core`), интеграционные проверяют их совместную работу и
воспроизводимость результатов на эталонных синтетических данных.
