========================================================================
src/configs/ — централизованная конфигурация проекта
========================================================================

Назначение
----------
Единый источник истины для всех общих констант, настроек и путей,
используемых в конвейере деконволюции масс-спектров ПОВ. Ранее эти
значения дублировались в разных модулях («магические» числа и строки);
теперь они хранятся в JSON-файлах и загружаются через тонкий загрузчик.

КРИТИЧЕСКИ ВАЖНО: значения научных констант (моноизотопные массы,
сдвиги дериватизации, пороги) не должны изменяться ни на одну значащую
цифру — любое расхождение ломает химическую валидность результатов.
При редактировании JSON сохраняйте точные значения.

Состав
------
  chemistry.json    Научные константы:
                      monoisotopic_masses  — моноизотопные массы элементов
                                             (H, C, N, O, S, P);
                      atomic_mass_elements — элементы и их порядок для
                                             назначения формул [M–H]⁻;
                      proton_mass          — масса протона (H⁺);
                      derivatization_shifts:
                        delta_cd3   = 17.03448 Da (COOH → COOCD3,
                                      дейтерометилирование);
                        delta_cd3co = 45.02939 Da (OH → OCOCD3,
                                      дейтероацилирование);
                      default_ion_mode     — режим ионизации ([M-H]-).

  pipeline.json     Параметры и пороги конвейера:
                      load_spectrum_defaults, default_brutto_dict,
                      formula_search, run_pipeline_defaults, test_mode,
                      thresholds, smoke_pipeline_params.

  paths.json        Относительные пути и имена файлов (data/, test_sets/,
                    имена спектров, выходной CSV, число тестовых наборов).
                    Абсолютные пути не хранятся.

  loader.py         Загрузчик: читает и кэширует JSON (load once),
                    проверяет наличие/корректность файла, даёт понятные
                    ошибки (ConfigError) и доступ к значениям как через
                    атрибуты, так и по ключу (ConfigNamespace). Пути к
                    JSON разрешаются относительно каталога модуля
                    (Path(__file__).parent), а не текущей рабочей папки,
                    поэтому конфигурация грузится одинаково независимо от
                    места запуска.

  __init__.py       Единая точка импорта.

Использование
-------------
    from src.configs import CHEM, PIPELINE, PATHS

    delta   = CHEM.derivatization_shifts["delta_cd3"]     # 17.03448
    masses  = CHEM.monoisotopic_masses                    # dict
    ppm_tol = PIPELINE.run_pipeline_defaults["ppm_tol"]   # 5.0
    data    = PATHS.data_dir                               # "data"

Динамический доступ по имени:

    from src.configs import load_config
    chem = load_config("chemistry")

Примечания
----------
* JSON поддерживает только строковые ключи. Диапазоны хранятся как
  списки [min, max]; модули, которым нужны кортежи (например
  DEFAULT_BRUTTO_DICT, FormulaSearchConfig.ranges), преобразуют их в
  tuple при загрузке, сохраняя исходные типы.
* Данные, специфичные для одного модуля и нигде не дублируемые
  (например ELEMENT_DATA в core/atoms.py с целочисленными ключами,
  библиотеки фрагментов в core/fragments.py), намеренно оставлены на
  местах — их вынос не даёт выигрыша и повышает риск.
