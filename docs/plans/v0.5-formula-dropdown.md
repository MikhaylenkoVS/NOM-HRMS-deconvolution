# План: выпадающий список формул в таблице результатов

> **Создан:** 2026-07-11 | **Обновлён:** 2026-07-12 | **Статус:** ✅ завершён (реализовано в v0.4.1)

## Суть

В колонке «Формула» таблицы результатов — Combobox с альтернативными кандидатами.
Автоматически подбирается лучший кандидат по критериям NOM-приоритизации.
Пользователь может вручную изменить формулу на любую из предложенных.
При смене формулы перестраиваются Van Krevelen и вкладка «Структуры».

## Шаги

| # | Задача | Статус |
|---|--------|--------|
| 1 | `assign_formulas`: сохранять всех кандидатов в колонку `all_candidates` | [x] |
| 2 | `build_result_table`: пробрасывать `all_candidates` в итоговую таблицу | [x] |
| 3 | `_fill_result_table`: диалог выбора формулы по двойному клику | [x] |
| 4 | `_on_candidate_selected`: обновление result_df + перестроение VK/Structures | [x] |
| 5 | Unit-тест: проверка структуры `all_candidates` | [x] |
| 6 | Интеграционный тест: candidates в итоговой таблице | [x] |

## Реализация (v0.4.1)

- `assign_formulas()` сохраняет список кандидатов в `table["all_candidates"]`
- `build_result_table()` пробрасывает `all_candidates` в выходной DataFrame
- Двойной клик по строке таблицы результатов → диалог `tk.Toplevel` со списком кандидатов
- При выборе новой формулы: обновление `result_df`, перестроение таблицы и графиков
- Поиск структур для нового пика запускается асинхронно с `first_only=True` и таймаутом 5с
- Визуальный индикатор в таблице: зелёный текст при единственном кандидате, жёлтый при множественных
