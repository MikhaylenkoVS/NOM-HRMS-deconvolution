# Структура тестовых данных и симуляций

Этот документ описывает, какие вспомогательные данные и скрипты используются для генерации тестовых наборов масс‑спектров и как они организованы в проекте.

## Общая идея

Текущая инфраструктура опирается на три основных компонента:

- `src/simulations` — код для генерации синтетических тестовых наборов (спектров и аннотаций), скопированный и адаптированный из подпроекта `generate_test_spectra/tools`.  
- `data/ref_data` — набор референсных молекул с минимальным набором дескрипторов, взятых из PubChem, а также вспомогательные визуализации (в частности, Van Krevelen plot).  
- `data/test_sets` — сгенерированные тест‑сеты (по одному в подпапку `set_XX`), которые используются как вход для алгоритма дешифровки (`core.py`) и как эталон для unit‑тестов.

Дополнительно в `tests/unit` лежат unit‑тесты, которые проверяют структурную валидность и внутреннюю согласованность сгенерированных данных.

---

## src/simulations

Папка `src/simulations` — это адаптированная копия исходной директории `generate_test_spectra/tools` подпроекта. Она содержит:

- скрипты генерации тестовых наборов (включая главный скрипт `generate_test_sets.py`);  
- вспомогательные функции для:
  - загрузки конфигурации (параметры централизованы в `src/configs/pipeline.json`);  
  - вычисления точных масс по формуле;  
  - применения дериватизации (deuteromethyl / deuteroacyl) с точными mass shift из `chemistry.json`;  
  - моделирования инструментальной ошибки (ppm‑ошибка) и шума в спектрах;  
  - записи результирующих CSV‑файлов (`original.csv`, `deutermethylated.csv`, `deuteroacylated.csv`, `annotations.csv`, `molecules.csv`).

Основной сценарий использования:

- изменение параметров генерации в `src/configs/pipeline.json` (секция `test_mode`: диапазон масс, модель ppm‑ошибки, интенсивность шума, mass shift при дериватизации, и т.п.);  
- запуск главного скрипта (`main()`), который последовательно обновляет все наборы в `data/test_sets`.

## tests/unit

Папка `tests/unit` — это копия и адаптация тестов из подпроекта `generate_test_spectra/tests`. Они обеспечивают базовую проверку корректности и согласованности тест‑данных.

Основные группы проверок:

1. **Структурная валидность файлов**  
   - Проверка наличия обязательных файлов в каждом наборе (`config.json`, `molecules.csv`, `original.csv`, `deutermethylated.csv`, `deuteroacylated.csv`, `annotations.csv`) для `set_01`.  
   - Проверка заголовков:
     - `molecules.csv` содержит ровно 14 колонок:  
       `set_id, compound_id, compound_number, name, smiles, inchi, formula, charge, mode, carboxyl_count, hydroxyl_count, other_fg, nom_like_flag, comment`.  
     - файлы спектров (`original.csv`, `deutermethylated.csv`, `deuteroacylated.csv`) имеют заголовок `mass,intensity`.  
     - `annotations.csv` имеет фиксированный заголовок с полями `mass_obs`, `mass_theor`, `mass_error_ppm`, `is_signal` и др.

2. **Согласованность config.json**  
   - Наличие обязательных ключей: `set_id`, `mass_range`, `ppm_error`, `noise`, `derivatization`.  
   - Внутри `ppm_error`: поля `type`, `mean`, `std`, `max_abs`.  
   - Внутри `noise`: `peak_count`, `intensity_fraction_max`.  
   - Внутри `derivatization`: блоки `deutermethyl` и `deuteroacyl` с параметрами, включая `mass_shift_per_group`.

3. **Консистентность molecules.csv**  
   - Для всех строк (если они есть) значения `carboxyl_count` и `hydroxyl_count` лежат в диапазоне `[1, 10]`.  

4. **Консистентность annotations и спектров**  
   - Для каждой строки в `annotations.csv`:
     - существует точный пик в соответствующем спектре (`original`, `deutermethylated`, `deuteroacylated`), у которого `mass` совпадает с `mass_obs` в пределах допуска, а `intensity` совпадает с точностью до заданного порога;  
     - `mass_error_ppm` согласован с формулой `((mass_obs - mass_theor) / mass_theor) * 1e6` и не выходит за пределы `max_abs` из `config["ppm_error"]`.

Эти тесты служат «охранным контуром» вокруг генератора данных: любое изменение в `src/simulations` или `config.json`, которое портит формат или внутреннюю согласованность, будет сразу поймано.

## data/ref_data

Папка `data/ref_data` содержит:

- **набор молекул** (файл или файлы CSV/TSV), подобранных по PubChem, с минимальным набором дескрипторов:
  - идентификатор (например, CID или собственный `compound_id`),  
  - эмпирическая формула,  
  - базовые структурные признаки (например, количество функциональных групп),  
  - возможные доп. поля для связи с основным массивом тестовых молекул.

- **`van_krevelen_nom_like.png`** — Van Krevelen plot (отношения H/C и O/C) для NOM‑подобных молекул.  
  Эта визуализация позволяет:
  - быстро оценить химическое пространство референсных молекул;  
  - сравнивать распределение тестовых молекул с реальной природной органикой;  
  - использовать рисунок как иллюстрацию в документации или статье.

Эти данные не участвуют напрямую в генерации тест‑спектров, но помогают калибровать и обосновывать выбор молекул и диапазонов параметров (массы, функциональные группы, и т.п.).

## data/test_sets

Папка `data/test_sets` содержит один или несколько наборов вида `set_01`, `set_02`, … Каждый набор представляет собой полностью самодостаточный комплект файлов, описывающих:

- исходные молекулы;  
- три спектра (original + два типа дериватизации);  
- аннотации и конфигурацию генерации.

### Структура одного набора (например, set_01)

Внутри `data/test_sets/set_01` лежат следующие файлы:

- Параметры генерации задаются централизованно в `src/configs/pipeline.json` (секция `test_mode`):
  - `load` — диапазоны масс для исходного и дериватизированных спектров;
  - `denoise` — параметры шумоподавления (force, intensity, quantile);
  - `assign` — режим назначения формул (`simple`), допустимая ppm-ошибка;
  - `series` — параметры поиска гомологических серий (ppm_tol, max_groups, allow_gaps);
  - `match_ppm` — допуск при сравнении результатов с эталоном.

- `molecules.csv` — список молекул для набора с минимальными метаданными:
  - 14 колонок:  
    `set_id, compound_id, compound_number, name, smiles, inchi, formula, charge, mode, carboxyl_count, hydroxyl_count, other_fg, nom_like_flag, comment`.  
  - mass здесь **не хранится** — теоретические массы вычисляются на лету по формуле.

- `original.csv` — «исходный» спектр:
  - две колонки: `mass,intensity`;  
  - `mass` — наблюдаемая масса с учётом ppm‑ошибки (mass_obs),  
  - `intensity` — интенсивность (для сигнала и шума).

- `deutermethylated.csv` — спектр после дейтерометилирования:
  - тот же формат `mass,intensity`;  
  - каждая молекула даёт несколько пиков с различной степенью дериватизации (от 0 до максимального числа целевых групп), плюс шум.

- `deuteroacylated.csv` — спектр после дейтероацилирования:
  - тот же формат `mass,intensity`;  
  - логика аналогична `deutermethylated.csv`, но нацелена на другую функциональную группу и другой mass shift.

- `annotations.csv` — полная аннотация всех пиков во всех трёх спектрах:
  - заголовок:  
    `set_id, spectrum_type, peak_id, mass_obs, intensity, mass_theor, exact_mass, mass_error_ppm, compound_id, compound_number, formula, derivatization_state, adduct_type, charge, assignment_confidence, is_signal`.  
  - `spectrum_type` — `original`, `deutermethylated` или `deuteroacylated`;  
  - `peak_id` — уникальный идентификатор пика внутри набора;  
  - `mass_obs` — наблюдаемая масса с учётом ppm‑ошибки (та же, что в соответствующем `*.csv`);  
  - `mass_theor` — теоретическая масса (без ошибки);  
  - `mass_error_ppm` — разница между `mass_obs` и `mass_theor` в ppm, укладывающаяся в `match_ppm` из `pipeline.json`;  
  - `compound_id`, `compound_number`, `formula` — идентификация молекулы для сигнальных пиков; для шумовых пиков эти поля пустые;  
  - `derivatization_state` — одно из значений (`none`, `deutermethyl`, `deuteroacyl`, промежуточные степени дериватизации и т.п.);  
  - `adduct_type`, `charge` — модель аддукта (на текущем этапе для отрицательного режима используется `[M-H]-`, `charge = -1`);  
  - `assignment_confidence` — уровень уверенности в назначении (на данном этапе чаще всего `high` для сигнала и `high`/технический флаг для шума);  
  - `is_signal` — булево: `True` для «настоящих» пиков (молекулы), `False` для искусственного шума.

## Использование тест‑сетoв для проверки алгоритма

Сгенерированные наборы в `data/test_sets` предназначены для:

- интеграционного тестирования конвейера (`src/core/pipeline.py`):  
  - подача `original.csv` / `deutermethylated.csv` / `deuteroacylated.csv` и `molecules.csv` на вход;  
  - сравнение результатов с `annotations.csv` (по `compound_id`, `derivatization_state`, `mass_obs`, допускам по ppm и т.п.);  

- unit‑тестов на уровень ниже (например, «функция сопоставления одного пика с молекулой»), с извлечением отдельных строк из `annotations.csv` и соответствующих пикoв из `*.csv`.
